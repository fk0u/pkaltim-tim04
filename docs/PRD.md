# Product Requirements Document (PRD)
**Project Name:** BorneoTrip
**Version:** 1.0 (MVP)
**Status:** In Development
**Last Updated:** Januari 2026

## 1. Executive Summary
BorneoTrip adalah platform agregator pariwisata yang didedikasikan untuk mempromosikan pariwisata berkelanjutan (sustainable tourism) di Kalimantan Timur. Platform ini tidak hanya menjual paket wisata, tetapi juga mengedukasi wisatawan tentang konservasi dan budaya lokal, serta memudahkan akses informasi event tahunan daerah.

## 2. Problem Statement
- **Fragmentasi Informasi:** Informasi mengenai event budaya (seperti Erau) dan paket wisata alam Kaltim tersebar di banyak website yang tidak standar.
- **Kurangnya Fokus Sustainability:** Banyak travel agent konvensional tidak memberikan transparansi mengenai dampak lingkungan atau kontribusi sosial dari trip mereka.
- **UX yang Kaku:** Website pariwisata pemerintah atau lokal seringkali memiliki antarmuka yang kuno dan tidak mobile-friendly.

## 3. Goals & Objectives
- **Integrasi Event & Wisata:** Menjadi "One-Stop Solution" untuk mencari event dan booking trip di Kaltim.
- **Promosi Ekowisata:** Meningkatkan awareness terhadap destinasi konservasi (Wehea, Derawan, Samboja).
- **Digitalisasi Ekonomi Lokal:** Membantu operator lokal mendapatkan pasar yang lebih luas melalui platform digital standar industri.

## 4. Key Features (MVP)

### 4.1. Core Experience
- **Smart Search Bar:** Pencarian berbasis Event, Lokasi, atau Paket.
- **Immersive Homepage:** Video background, Horizontal scroll untuk event, dan kartu paket wisata visual.

### 4.2. Booking System
- **Detailed Package Page:** Itinerary harian, fasilitas, dan transparansi harga.
- **Interactive Booking Card:** Sticky price calculator, date picker, dan guest counter.
- **Checkout Flow:** Form pengisian data diri, pemilihan metode pembayaran (simulasi), dan E-Voucher generation.

### 4.3. User Accounts (Traveler)
- **Client Dashboard:** Halaman profil yang menampilkan "Active Trip", riwayat, dan gamifikasi (Level XP).
- **Gamification:** Sistem poin/level untuk memberi reward kepada traveler berkelanjutan.

### 4.4. Administration (Operator/Admin)
- **Admin Dashboard:** Overview pendapatan, jumlah booking, dan user baru.
- **Booking Management:** Tabel interaktif untuk Konfirmasi/Cancel pesanan.
- **Customer CRM:** Database pelanggan dengan detail spending dan level.

## 5. Non-Functional Requirements
- **Performance:** Load time halaman utama < 2 detik.
- **Responsiveness:** UI harus 100% fungsional di Mobile, Tablet, dan Desktop.
- **Usability:** Feedback sistem (Toast/Modal) untuk setiap interaksi penting.
- **Visual:** Design language modern (Glassmorphism, Large Typography, Green/Emerald Color Palette) untuk mencerminkan "Premium Nature".

## 6. Success Metrics
- **Conversion Rate:** % Pengunjung yang mengklik "Pesan Sekarang".
- **Engagement:** Rata-rata waktu yang dihabiskan di halaman detail paket.
- **Retention:** % User yang kembali login ke Dashboard.

## 7. Future Roadmap (Post-MVP)
- Integrasi Payment Gateway nyata (Midtrans/Xendit).
- Fitur "Carbon Calculator" untuk menghitung jejak karbon perjalanan.
- Integrasi Peta Interaktif Real-time.
- Multi-vendor system (Open marketplace untuk guide lokal).
